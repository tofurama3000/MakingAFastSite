Computer Code

At times we want to format computer code, or keyboard input and computer output. This is great for technical information, such as how to login to a computer, how to setup an Arduino, or even tutorials on how to make a website.

There are several tags, which tend to give the same formatting result in most browsers (which is why people will use them interchangeably). However, browsers may display them differently, and some might even do special things to try to be helpful. For that reason, all of them will be listed here.

Below is a list of these tags and what they're used for:

  • <kbd> - This is for Keyboard Input
  • <samp> - This is for Sample Computer Output
  • <code> - This is for computer code

Preformatted Text

Preformatted text is text that is already formatted with whitespace. This means that it will keep all spaces and line breaks that are in the inside text, whereas in normal HTML line breaks are ignored and extra whitespace is removed. It is done by using the "<pre>" tag, and using the "</pre>" end tag to define where to stop. This is often used to wrap the tags in the above section since you don't have to manually specify line breaks - you just add a newline. It can be used on its own, and will often be formatted similarly to the above code. In fact, all code examples use this tag! Below is an example: 

<pre>
This   text
Is Preformatted
     Using the
pre tag
</pre>

Please note that tags inside will still be evaluated, so if you wrap a word with the <b> tag it will bold the text, like so: 

This preformatted text has
    BOLD text

Marking Variables

In math and code, we often have variables. To tell the computer that something is a variable, we use the <var> tag. This tag will tell the browser to format the variable. The formatting can depend on the browser. Below is an example 

5<var>x</var> + 3 = 28
